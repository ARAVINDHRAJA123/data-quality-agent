## Root Cause

3 rows in `raw.bank_transactions` were loaded with a **NULL narration** field. The merchant name in this pipeline is not stored directly in the raw source — it is **derived** from the narration string in the staging model (`stg_bank__transactions`) using a REGEXP/COALESCE expression. When the narration is NULL, the merchant extraction returns NULL, which propagates unchanged through `int_transactions_categorised` into `fct_transactions`, causing the `not_null_merchant` test to fail.

The bad rows were loaded on **2026-06-27** with `bank_name = TEST` and reference numbers `TEST001`, `TEST002`, `TEST003`, indicating a test or misconfigured ingestion run that bypassed the narration validation in `load_to_bigquery.py`.
